Week 7: Binary Search and Efficient Sorting

CS 301 — Algorithm Design & Analysis · Prof. R. Knuth · 45 min read

1. Binary Search

Binary search is a divide and conquer algorithm that efficiently locates a target value in a sorted array. Instead of examining every element, it repeatedly halves the search space, achieving O(log n) time complexity.

The key invariant: at every step, if the target exists in the array, it must lie within the current [low, high] window.

Algorithm

def binary_search(arr, target):
    low, high = 0, len(arr) - 1

    while low <= high:
        mid = (low + high) // 2          # integer division avoids overflow

        if arr[mid] == target:
            return mid                   # found
        elif arr[mid] < target:
            low = mid + 1                # target in right half
        else:
            high = mid - 1              # target in left half

    return -1                           # not found
⚠ Common mistake: Using mid = (low + high) / 2 with integer overflow in C/Java. The safe form is mid = low + (high - low) // 2.

2. Comparison-Based Sorting

A fundamental result in algorithm theory is that any comparison-based sort requires at least O(n log n) comparisons in the worst case. This is the theoretical lower bound, proven via a decision-tree argument.

Merge Sort

Merge sort is a stable, O(n log n) algorithm based on the divide and conquer paradigm. It splits the array in half, sorts each half recursively, then merges them in linear time. 

def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left  = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    return result + left[i:] + right[j:]

Quick Sort

Quick sort selects a pivot element and partitions the array into elements less than, equal to, and greater than the pivot. In practice it is often faster than merge sort due to better cache locality, despite its O(n²) worst-case time complexity.

3. Big O Notation Recap

Big O notation describes the upper bound of an algorithm's growth rate. We drop constant factors and lower-order terms.

O(1)ConstantArray index lookup

💡 Study tip: When you see a while-loop that halves its search space each iteration, the complexity is almost always O(log n). This is the hallmark of binary search and balanced BST operations.

4. Introduction to Dynamic Programming

Dynamic programming (DP) solves complex problems by breaking them into overlapping sub-problems and storing intermediate results (memoization or tabulation) to avoid redundant computation.

A DP solution is applicable when the problem has:

  • Optimal substructure — the optimal solution contains optimal sub-solutions
  • Overlapping subproblems — the same sub-problems are solved multiple times

Example: Fibonacci with Memoization

def fib(n, memo={}):
    if n in memo: return memo[n]
    if n <= 1:   return n
    memo[n] = fib(n-1, memo) + fib(n-2, memo)
    return memo[n]

Without memoization: O(2ⁿ). With memoization: O(n) time, O(n) space. This dramatic improvement illustrates why dynamic programming is a powerful technique for algorithm design.
